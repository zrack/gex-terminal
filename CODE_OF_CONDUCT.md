# Code of Conduct

## Our Pledge

`gex-terminal` is a market research and engineering project. We want it to be a
welcoming place for traders, developers, data engineers, and researchers who are
learning in public and improving the tool together.

Participants are expected to be respectful, curious, and constructive.

## Expected Behavior

- Be kind and professional, especially when disagreeing.
- Assume good intent, but be willing to clarify impact.
- Keep feedback specific, actionable, and grounded in the project.
- Respect different levels of trading, Python, market-data, and open-source
  experience.
- Protect private information, including credentials, account IDs, logs, and
  paid market-data payloads.
- Clearly label financial-market assumptions, limitations, and uncertainty.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or conduct.
- Publishing private credentials, account details, or proprietary market-data
  payloads.
- Repeatedly derailing issues, pull requests, or discussions.
- Presenting research output as financial advice.
- Spam, scams, referral abuse, or promotional behavior unrelated to the project.

## Reporting

If you see behavior that violates this code of conduct, contact the maintainers
through the repository's GitHub issue or discussion channels. If the report
contains private information, share only the minimum public detail needed and ask
for a private follow-up path.

Maintainers will review reports in context and may remove comments, close
threads, restrict participation, or take other moderation steps needed to keep
the project healthy.

## Scope

This code of conduct applies to project spaces, including GitHub issues, pull
requests, discussions, documentation, examples, and community conversations
directly related to `gex-terminal`.
